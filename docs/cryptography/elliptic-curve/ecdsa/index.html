ECDSA (Elliptic Curve Digital Signature Algorithm)

1. Background: What is ECDSA?‚Äã

ECDSA is a digital signature algorithm based on Elliptic Curve Cryptography (ECC).

  • It‚Äôs used to sign messages (prove authenticity) and verify signatures (prove origin & integrity).
  • Widely used in Bitcoin, Ethereum, TLS certificates, SSH, JWTs, etc.
  • Its security is based on the hardness of the Elliptic Curve Discrete Logarithm Problem (ECDLP).

2. Key Concepts You Need First‚Äã

(a) Finite Fields‚Äã

Cryptography works over finite fields (like modular arithmetic). For example:

In mod 7:
  3 + 5 ‚â° 1 (mod 7)
  3 √ó 5 ‚â° 1 (mod 7)

(b) Elliptic Curves‚Äã

An elliptic curve is defined by an equation like:

Demo: Elliptic Curves

y¬≤ = x¬≥ + ax + b (mod p)

where p is a prime (so we‚Äôre in a finite field).

These curves aren‚Äôt circles/ellipses ‚Äî instead, they form a special set of points with useful math properties.

(c) Group Law (Point Addition)‚Äã

On elliptic curves, we define:

  • Point addition: adding two points gives another point on the curve.
  • Scalar multiplication: repeating addition many times (like multiplication in normal math).

This operation is easy one way, but hard to reverse ‚Äî the basis of ECC security.

3. ECDSA Key Generation‚Äã

  1. Choose a standard elliptic curve (like secp256k1 in Bitcoin).
  2. Pick a random private key d (a number in [1, n-1], where n is the order of the curve).
  3. Compute public key Q = d √ó G, where G is the generator point on the curve.

So:

  • Private key = number d
  • Public key = point Q

4. Signing a Message‚Äã

To sign a message m with private key d:

  1. Compute hash: z = HASH(m) (SHA-256, for example).

  2. Pick a random number k (nonce, must be secret & unique).

  3. Compute point: R = k √ó G, let r = R.x mod n.

    • If r = 0, restart.

  4. Compute: s = k‚Åª¬π (z + r¬∑d) mod n.

    • If s = 0, restart.

  5. Signature = pair (r, s).

5. Verifying a Signature‚Äã

To verify (r, s) against message m and public key Q:

  1. Check r, s are in range [1, n-1].

  2. Compute hash: z = HASH(m).

  3. Compute inverse: w = s‚Åª¬π mod n.

  4. Compute:

    • u1=zwmod‚Äâ‚Äânu_1 = zw \mod nu1‚Äã=zwmodn
    • u2=rwmod‚Äâ‚Äânu_2 = rw \mod nu2‚Äã=rwmodn
    • X=u1G+u2QX = u_1 G + u_2 QX=u1‚ÄãG+u2‚ÄãQ.

    Let x = X.x mod n.

  5. Signature valid if x = r.

üîë Key weakness: If kkk is ever reused or leaked ‚Üí private key ddd can be recovered (this happened in Bitcoin/PS3 hacks).

6. Why ECDSA Works‚Äã

  • Signing equation: s = k‚Åª¬π(z + r¬∑d)
  • Verification recombines G and Q so that the only way r matches is if the signer knew d.
  • Security: Breaking ECDSA means solving ECDLP, which is computationally infeasible.

Step 2. Modular Inverse (needed for division)‚Äã

We‚Äôll use the extended Euclidean algorithm to compute inverses mod p.

def modinv(k, p):
    """Modular inverse using Extended Euclidean Algorithm"""
    if k == 0:
        raise ZeroDivisionError("division by zero")
    return pow(k, -1, p)  # Python 3.8+ supports this

Step 3. Point Addition on the Curve‚Äã

We need point addition and doubling.

def point_add(P, Q):
    if P is None: return Q
    if Q is None: return P
    
    (x1, y1) = P
    (x2, y2) = Q
    
    if x1 == x2 and (y1 != y2 or y1 == 0):
        return None  # point at infinity
    
    if P == Q:
        # slope = (3*x1^2 + a) / (2*y1)
        m = (3 * x1 * x1 + a) * modinv(2 * y1, p) % p
    else:
        # slope = (y2 - y1) / (x2 - x1)
        m = (y2 - y1) * modinv(x2 - x1, p) % p
    
    x3 = (m * m - x1 - x2) % p
    y3 = (m * (x1 - x3) - y1) % p
    
    return (x3, y3)

Step 4. Scalar Multiplication

Multiply a point by an integer (repeated addition).

def scalar_mult(k, P):
    R = None
    while k > 0:
        if k & 1:
            R = point_add(R, P)
        P = point_add(P, P)
        k >>= 1
    return R

Step 5. Key Generation

Pick random private key ddd, compute public key Q=d‚ãÖGQ = d¬∑GQ=d‚ãÖG.

import random

def generate_keypair():
    d = random.randrange(1, n)   # private key
    Q = scalar_mult(d, G)        # public key

Step 6. ECDSA Signing

Follow the ECDSA signing steps.

def sign(m, d):
    z = m % n  # pretend "hash" is just m mod n
    while True:
        k = random.randrange(1, n)
        R = scalar_mult(k, G)
        r = R[0] % n
        if r == 0:
            continue
        s = (modinv(k, n) * (z + r * d)) % n
        if s == 0:
            continue
        return (r, s)

Step 7. ECDSA Verification

Verify a signature.

def verify(m, sig, Q):
    r, s = sig
    if not (1 <= r < n and 1 <= s < n):
        return False
    z = m % n
    w = modinv(s, n)
    u1 = (z * w) % n
    u2 = (r * w) % n
    X = point_add(scalar_mult(u1, G), scalar_mult(u2, Q))
    if X is None:
        return False
    return (X[0] % n) == r

3. Schnorr Signatures (cleaner alternative)

Schnorr is mathematically simpler and has nice properties (linear, provably secure under discrete log).

Signing‚Äã

  1. Hash the message: z=H(m)z = H(m)z=H(m).
  2. Choose random nonce kkk.
  3. Compute R=kGR = kGR=kG.
  4. Compute challenge e=H(R‚à•Q‚à•z)e = H(R \parallel Q \parallel z)e=H(R‚à•Q‚à•z).
  5. Compute response s=k+edmod‚Äâ‚Äâns = k + ed \mod ns=k+edmodn.
  6. Signature is (R,s)(R, s)(R,s).

Verifying‚Äã

Given signature (R,s)(R, s)(R,s):

  1. Compute e=H(R‚à•Q‚à•z)e = H(R \parallel Q \parallel z)e=H(R‚à•Q‚à•z).

  2. Check if sG=R+eQsG = R + eQsG=R+eQ.

    • If true, signature is valid.

üîë Advantages over ECDSA:

  • Simpler math.
  • No fragile dependency on k‚àí1k^{-1}k‚àí1.
  • Supports batch verification (verify many sigs at once).
  • Enables signature aggregation (combine many sigs into one). ‚Üí This is why Bitcoin Taproot adopted Schnorr.
